1. Introduction

This project implements a high-performance, web-based simulator and automated solver engine for the classic Lights Out puzzle. The puzzle consists of a grid of lights (cells), each of which can be either on (1) or off (0). Clicking any cell toggles its state and the states of its four adjacent neighbors. The objective is to transition the entire board from its initial configuration to a state where all lights are turned off.

While the puzzle is simple to play, finding an optimal solution sequence is a well-known problem in mathematical optimization. This project uses Linear Algebra over the Galois Field of order 2, GF(2), to solve the puzzle. In GF(2), arithmetic operations are performed modulo 2, meaning addition corresponds directly to the bitwise Exclusive OR (XOR) operation.

The simulation engine, interactive user interface, and algebraic solvers are written in vanilla **JavaScript**, **HTML5**, and **CSS3**, producing a web application with real-time feedback, step-by-step auto-solves, hint generation, and procedurally generated puzzles across three difficulty tiers.

3. Mathematical Foundations: Linear Algebra over GF(2)

A Lights Out grid with $N$ rows and $N$ columns has $M = N^2$ cells. The state of the board can be represented as a vector $\vec{b} \in \mathbb{F}_2^M$, where each component $b_i \in \{0, 1\}$ represents whether cell $i$ is off or on.

Pressing a cell $j$ toggles a specific set of cells (cell $j$ itself and its cardinal neighbors). The effect of clicking cell $j$ can be modeled as a vector $\vec{a}_j \in \mathbb{F}_2^M$, where $a_{ij} = 1$ if clicking cell $j$ toggles cell $i$, and $a_{ij} = 0$ otherwise.

Linear System Modulo 2 Let $x_j \in \{0, 1\}$ denote whether cell $j$ is clicked ($x_j = 1$) or not clicked ($x_j = 0$). Since clicking a cell twice returns it to its original state, we only need to decide if we click each cell once or not.

The final state of the board $\vec{s}_{final}$ after a sequence of clicks $\vec{x}$ is: $$\vec{s}_{final} = \vec{s}_{initial} \oplus \bigoplus_{j=1}^{M} x_j \vec{a}_j$$ To solve the puzzle, we want $\vec{s}_{final} = \vec{0}$, which translates to: $$\mathbf{A}\vec{x} \equiv \vec{s}_{initial} \pmod 2$$ where $\mathbf{A} \in \mathbb{F}_2^{M \times M}$ is the **Toggle Matrix** whose columns are the toggle vectors $\vec{a}_j$.

Since addition in $\mathbb{F}_2$ is equivalent to XOR and multiplication is equivalent to AND, this is a standard system of linear equations $\mathbf{A}\vec{x} = \vec{y}$ where all arithmetic is carried out using XOR and AND logic gates.

5. Algorithm Design & Gaussian Elimination

To find the winning sequence $\vec{x}$, the solver performs **Gaussian Elimination** on the augmented matrix $[\mathbf{A} \mid \vec{b}]$ over $\mathbb{F}_2$. The steps are:

  1. **Augmentation:** Build the $M \times (M + 1)$ matrix where the first $M$ columns represent the toggle rules and the last column represents the current board state.
  2. **Forward Elimination:** Reduce the matrix to row echelon form using XOR operations for row additions. Since pivot values can only be 0 or 1, row reduction involves checking if the cell at the pivot coordinate is 1, and XORing subsequent rows that have 1 in the same column.
  3. **Solvability Check:** If any row has zeros in the first $M$ columns but a 1 in the augmented column, no solution exists.
  4. **Back Substitution:** Extract the variable assignments. Free variables are defaulted to 0 to yield the minimum-click solution.
// Gaussian Elimination over GF(2)
function solveGF2(boardState) {
    const A = buildToggleMatrix();
    const aug = [];
    for (let i = 0; i < TOTAL; i++) {
        aug[i] = new Uint8Array(TOTAL + 1);
        for (let j = 0; j < TOTAL; j++) aug[i][j] = A[i][j];
        aug[i][TOTAL] = boardState[i];
    }
    
    let rank = 0;
    const pivotCol = new Int8Array(TOTAL).fill(-1);

    for (let col = 0; col < TOTAL; col++) {
        let pivotRow = -1;
        for (let row = rank; row < TOTAL; row++) {
            if (aug[row][col] === 1) {
                pivotRow = row;
                break;
            }
        }
        if (pivotRow === -1) continue;

        // Swap rows
        [aug[rank], aug[pivotRow]] = [aug[pivotRow], aug[rank]];
        pivotCol[col] = rank;

        // Eliminate column in other rows using XOR
        for (let row = 0; row < TOTAL; row++) {
            if (row !== rank && aug[row][col] === 1) {
                for (let k = 0; k <= TOTAL; k++) {
                    aug[row][k] ^= aug[rank][k];
                }
            }
        }
        rank++;
    }
    // Reconstruct and return minimum-weight vector...
}

6. Procedural Puzzle Generation & Matrix Rank Filtering

An arbitrary initial board state is not guaranteed to be solvable. For a standard 5×5 board, the rank of the toggle matrix $\mathbf{A}$ is 23 (not 25). This implies:

  • The kernel (null space) of $\mathbf{A}$ has dimension $\dim(\ker(\mathbf{A})) = 25 - 23 = 2$.
  • There are $2^2 = 4$ configurations that can be clicked without changing any lights, which correspond to the null vectors of the matrix.
  • Only $2^{23}$ of the $2^{25}$ possible configurations are solvable. This means only 25% of randomly initialized boards can be solved.
Rank Filtering vs. Retrograde Step Generation To guarantee 100% solvability, the generator begins with an all-off board ($\vec{0}$) and performs a backward walk. It selects a set number of unique cells to toggle based on the difficulty tier, ensuring the target state is mathematically valid and resides within the image of the toggle operator.

9. Conclusion

This project demonstrates how linear algebra can be applied to solve grid-based puzzles. By representing state transitions as system operations over $\mathbb{F}_2$, we bypassed traditional heuristic search algorithms and solved the puzzle in polynomial time.

The modular JavaScript implementation ensures smooth rendering performance, while matrix rank evaluations guarantee a bug-free experience by preventing unsolvable board states.
